# Workflows

How to compose skills from across the marketplace into an end-to-end loop. These are menus, not pipelines — skip steps that don't apply, and invoke skills out of order whenever the work demands it.

| Workflow | When to use |
|----------|-------------|
| [Greenfield project](greenfield-project.md) | Brand-new project — no code, no backlog, maybe just an idea. |
| [Existing project](existing-project.md) | Feature, fix, or chore on a live codebase. |

Each workflow doc lists the **skill sequence**, **decision points** (when to skip a step, loop back, or branch), and **artifacts produced** at each step.
