# Update Workflow

Revise specific sections of an existing `DESIGN.md` to reflect changes.

## Steps

1. Read the existing document
2. Define update scope
3. Explore and revise
4. Regenerate preview, verify, and present

## Step 1: Read Existing Document

Read `DESIGN.md` at the project root in full.

## Step 2: Define Scope

If the user specifies the changes (e.g., "we added a new accent color"):
  → Use their description directly.

If the user says "update it" without specifics:
  → Ask what changed or what prompted the update.

Summarize scope before proceeding: "I'll update these sections: [list]. Sound right?"

## Step 3: Explore and Revise

1. Explore the codebase for changes in the scoped areas.
2. Edit only affected sections — preserve unchanged sections verbatim.
3. Match the existing writing style and level of detail.
4. Update the "Last updated" date.

## Step 4: Regenerate Preview, Verify, and Present

1. Re-read modified sections in the context of surrounding sections to check coherence.
2. Confirm cross-references still hold.
3. Regenerate the visual preview (see [SKILL.md → Output](../SKILL.md#output) for the command).
4. Summarize what changed and present `DESIGN.md` and `DESIGN-preview.html` together.
