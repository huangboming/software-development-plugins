# Release Notes Template

Audience is end users. Every entry answers "what can I now do, avoid, or expect?" — never restate the commit.

````markdown
# <Product Name> — v<version>

> Released: <YYYY-MM-DD>

## Highlights

- **<Highlight>** — <one-sentence user benefit>

## Breaking Changes

- **<Change>** — <who is affected>

  **Migration:** <specific steps>

## What's New

- <User-benefit framing>

## Bug Fixes

- <Name the symptom that was resolved>
````

Drop any section with no entries. If there are <5 total entries, drop the section headings and ship a flat list.
